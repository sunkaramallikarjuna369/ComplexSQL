üìò WHAT is INNER JOIN?

INNER JOIN returns only the rows where there is a match in both tables based on the join condition. If a row in either table doesn't have a matching row in the other table, it is excluded from the result set.

INNER JOIN Visualization

Blue: Rows only in Table A (excluded)

Red: Rows only in Table B (excluded)

Purple: Matching rows (INNER JOIN result)

üéØ WHY use INNER JOIN?
  • Data Integrity: Only returns complete, matched data from both tables
  • Performance: Generally faster than OUTER JOINs as it processes fewer rows
  • Clean Results: No NULL values from missing matches
  • Common Use Case: Most frequently used join type in SQL queries
üí°
Pro Tip: INNER JOIN is the default join type. Writing just "JOIN" without specifying the type is equivalent to "INNER JOIN".

üîß HOW to use INNER JOIN?
INNER JOIN - All RDBMS Universal 
-- Basic INNER JOIN syntax
SELECT 
    e.employee_id,
    e.first_name,
    e.last_name,
    d.department_name
FROM employees e
INNER JOIN departments d 
    ON e.department_id = d.department_id;

-- Equivalent using just JOIN (INNER is default)
SELECT 
    e.employee_id,
    e.first_name,
    d.department_name
FROM employees e
JOIN departments d 
    ON e.department_id = d.department_id;

-- Multiple table INNER JOIN
SELECT 
    e.first_name,
    d.department_name,
    p.project_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id
INNER JOIN projects p ON e.employee_id = p.lead_id;

-- INNER JOIN with additional conditions
SELECT 
    e.first_name,
    e.salary,
    d.department_name
FROM employees e
INNER JOIN departments d 
    ON e.department_id = d.department_id
WHERE e.salary > 50000
ORDER BY e.salary DESC;
